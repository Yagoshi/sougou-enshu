# チーム開発セットアップ手順書

## 前提
- GitHubからの招待メールのリンクをクリックして、リポジトリへの参加を承認済みであること
- Git、Python、MySQLがインストール済みであること

---

## Step 1：リポジトリをクローンする

GitBashを開いて、作業フォルダに移動してからクローンしてください。

```bash
cd C:/Users/自分のユーザー名/Desktop/Workspaces/総合演習
git clone https://github.com/Yagoshi/sougou-enshu.git
cd sougou-enshu
```

---

## Step 2：developブランチに切り替える

```bash
git checkout develop
```

以下のように表示されればOKです。

```
Switched to branch 'develop'
Your branch is up to date with 'origin/develop'.
```

---

## Step 3：.envファイルを作成する

`.env.example` をコピーして `.env` を作成します。

```bash
cp .env.example .env
```

---

## Step 4：.envファイルを編集する

VS Codeで `.env` を開いて、`DB_NAME` の値を自分が作成するDB名に書き換えてください。

```
DB_NAME=sougou_enshu
```

> ⚠️ `.env` はGit管理対象外です。絶対にコミットしないでください。

---

## Step 5：MySQLでデータベースを作成する

GitBashからMySQLにログインしてDBを作成します。

```bash
mysql -u root -p
```

パスワードを入力してログインしたら、以下のコマンドを実行してください。

```sql
CREATE DATABASE `sougou-enshu`;
exit;
```

> ⚠️ DB名にハイフン（`-`）が含まれる場合はバッククォートで囲む必要があります。

---

## Step 6：必要なパッケージをインストールする

```bash
pip install pymysql python-dotenv django
```

> ℹ️ `mysqlclient` ではなく `pymysql` を使っています。`mysqlclient` はWindowsでのインストールにコンパイラが必要なため、`pymysql` を採用しています。

---

## Step 7：マイグレーションを実行する

```bash
python manage.py migrate
```

エラーなく完了すればOKです。

---

## Step 8：開発サーバーを起動して動作確認する

```bash
python manage.py runserver
```

ブラウザで http://127.0.0.1:8000 を開いてトップ画面が表示されればセットアップ完了です。

---

## Step 9：開発作業のブランチ運用（GitFlow）

### 基本ルール

| ブランチ | 用途 |
|----------|------|
| `main` | リリース用。直接pushは禁止 |
| `develop` | 開発のベース。直接pushは禁止 |
| `feature/〇〇` | 機能追加はここで作業 |
| `bugfix/fix-〇〇` | バグ修正はここで作業 |

### ブランチ名の例

| 種類 | 例 |
|------|----|
| 機能追加 | `feature/add-payment-api` |
| 機能追加 | `feature/add-image-upload` |
| 機能追加 | `feature/modify-design` |
| バグ修正 | `bugfix/fix-login-error` |
| バグ修正 | `bugfix/fix-cart-amount` |

---

### 作業の流れ（毎回この手順で進める）

---

#### ① developの最新を取得する

作業を始める前に必ずdevelopの最新を取得してください。

```bash
git checkout develop
git pull origin develop
```

以下のように表示されればOKです。

```
Already up to date.
```

> ⚠️ この手順を忘れると、他の班員の変更が取り込まれていない古い状態でブランチを切ることになります。

---

#### ② 作業ブランチを切り出す

`feature/` に続けて、何をするか分かる名前をつけてください。

```bash
git checkout -b feature/add-payment-api
```

以下のように表示されればOKです。

```
Switched to a new branch 'feature/add-payment-api'
```

今いるブランチを確認したい場合は以下を実行してください。

```bash
git branch
```

`*` がついているブランチが現在いるブランチです。

```
  develop
* feature/add-payment-api
  main
```

---

#### ③ ファイルを編集する

VS Codeで作業を行ってください。

編集中に現在の変更状態を確認したい場合は以下を実行してください。

```bash
git status
```

---

#### ④ 変更をステージングする

編集が終わったら変更をステージングします。

全ファイルをまとめてステージングする場合は以下を実行してください。

```bash
git add .
```

特定のファイルだけステージングしたい場合は以下のようにファイル名を指定してください。

```bash
git add store/views.py
```

ステージング後に確認したい場合は以下を実行してください。

```bash
git status
```

`Changes to be committed:` の下に緑色でファイル名が表示されていればOKです。

---

#### ⑤ コミットする

```bash
git commit -m "支払API連携の実装"
```

コミットメッセージは **何をしたか** が分かるように書いてください。

以下のように表示されればOKです。

```
[feature/add-payment-api xxxxxxx] 支払API連携の実装
 1 file changed, 20 insertions(+), 2 deletions(-)
```

---

#### ⑥ リモートにpushする

```bash
git push origin feature/add-payment-api
```

以下のように表示されればOKです。

```
remote: Create a pull request for 'feature/add-payment-api' on GitHub by visiting:
remote:      https://github.com/Yagoshi/sougou-enshu/pull/new/feature/add-payment-api
To https://github.com/Yagoshi/sougou-enshu.git
 * [new branch]      feature/add-payment-api -> feature/add-payment-api
```

---

#### ⑦ GitHubでプルリクエストを作成する

1. https://github.com/Yagoshi/sougou-enshu を開く
2. 「Compare & pull request」ボタンをクリック
3. マージ先が `develop` になっていることを確認する

> ⚠️ `main` になっていたら必ず `develop` に変更してください。

4. タイトルに何をしたか書く（例：`支払API連携の実装`）
5. 必要に応じて説明欄に詳細を書く
6. 「Create pull request」をクリック

---

#### ⑧ 班員にレビューしてもらう

レビュアー（他の班員）は以下を確認してください。

- バグがないか
- 命名が適切か
- 不要なファイルが含まれていないか

問題なければ「Approve」をクリックしてください。

---

#### ⑨ 全員承認後にマージする

1. 全員がApproveしたら「Merge pull request」をクリック
2. 「Confirm merge」をクリック
3. マージ後、作業ブランチは「Delete branch」で削除してOK

---

#### ⑩ マージ後にローカルのdevelopを更新する

マージが完了したら、全員がローカルのdevelopを最新状態に更新してください。

```bash
git checkout develop
git pull origin develop
```

---

## 困ったときの確認コマンド

```bash
# 今いるブランチを確認
git branch

# リモートの最新を取得
git pull origin develop

# 変更状態を確認
git status

# 直近のコミット履歴を確認
git log --oneline -5
```
